# Plan: Preserve Rejected Upload Records in pending_uploads Table

## Goal
Address the user's concern about rejected records being removed from the pending_uploads table. Instead of deleting rejected uploads, update their status to 'rejected' to maintain a history while hiding them from the pending list.

## Current Behavior
- Currently, rejectUpload() deletes rejected records from pending_uploads and removes the file from storage.
- This removes the record entirely, which may be undesirable for auditing or history purposes.

## Proposed Changes
1. **Modify rejectUpload() Function**:
   - Change from deleting the record to updating status = 'rejected'.
   - Keep file deletion from storage (since rejected content shouldn't be accessible).
   - Update alert message to reflect status change.

2. **Ensure loadPendingUploads() Filters Correctly**:
   - Confirm it only shows status = 'pending'.
   - Rejected uploads (status = 'rejected') will not appear in the admin panel.

3. **Database Schema Confirmation**:
   - The pending_uploads table already has a status column (default 'pending').
   - No schema changes needed.

## Implementation Steps
1. Update rejectUpload() in admin.html to use update() instead of delete().
2. Update rejectUpload() in coadmin.html to match.
3. Test that rejected uploads disappear from pending list but remain in database.

## Benefits
- Maintains audit trail of all upload attempts.
- Prevents data loss.
- Admins can still see rejected uploads if needed (e.g., by querying status = 'rejected').

## Risks
- Table may grow over time with rejected records.
- Consider periodic cleanup of old rejected records if needed.

## Next Steps
Confirm this approach aligns with user's intent, then proceed with implementation.
